你必须遵循下述的“风格指南”。你构建的所有页面和组件都必须体现这种风格。最终交付的界面应该给人一种专业、克制、功能驱动的感觉。

风格指南：企业级高对比度简约风格

在构建所有页面时，请严格遵守以下设计原则：
1. 整体布局与空间感
* 背景与内容区分： 页面主体背景应为非常浅的灰色（例如 bg-muted）。所有内容（表单、数据、设置）都必须被封装在白色的 Card 组件中。
* 呼吸感： 布局必须是宽松且"不拥挤"的。在元素之间、卡片之间以及页面边缘使用充足的留白（padding 和 margin）。页面内容区域使用 `space-y-6` 来分隔多个 Card 组件。
* 页面容器： 使用 PageContainer 组件包裹页面内容，提供统一的页面标题和布局结构。
2. 核心色板：中性与高对比度
* 主色调： UI 的基础色调是黑色、白色和不同深浅的灰色。
* 交互色：避免使用明亮的“品牌蓝”或“成功绿”。
    * 主要操作： 所有主要的、引导性的操作（例如"下一步"、"保存"、"添加"）的 Button 应使用深色背景和白色文字，可通过 `className="bg-black text-white"` 显式指定，或确保主题配置中的 primary 颜色为深色。这提供了强烈的高对比度。
    * 次要操作： 次要操作（例如“上一步”、“取消”、“保存草稿”）应使用 variant="outline"（轮廓按钮）。
    * 危险操作： 危险操作（如“删除”）可谨慎使用 variant="ghost" 或 outline 搭配红色图标/文字。
3. 整体美感
* 专业与克制： 界面不应"有趣"或"花哨"。它的美感来源于其功能性、清晰的层次结构和一致的组件应用。
* 功能驱动： 每一个 UI 元素的存在都应有明确的功能目的。避免纯粹的装饰性元素。
4. 列表与表格交互
* 动作列按钮： 列表页面的动作列（操作列）中的按钮必须使用文字标签，而非图标。例如使用 `{t("module.viewDetail")}` 而不是 `<Eye className="h-4 w-4" />`。这确保了操作的明确性和可访问性。
5. 状态标签样式
* 统一规范： 状态标签应使用统一的样式。推荐使用药丸形状（rounded-full）的自定义标签组件，包含边框，使用小号文字（text-xs font-medium）。也可使用 Badge 组件，但需保持样式一致性。根据状态使用对应的颜色方案：待处理/进行中使用黄色系（bg-yellow-100 text-yellow-800），已完成/已批准使用绿色系（bg-green-100 text-green-800），已取消/失败使用红色系（bg-red-100 text-red-800），默认/未知使用灰色系（bg-gray-100 text-gray-800）。
6. 搜索表单布局
* 统一规范： 列表页面的搜索条件应封装在独立的 Card 中，使用响应式网格布局（grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4 items-end）。每个输入框用 div 包裹，搜索和清空按钮放在最后一个网格项中，使用 flex gap-2 布局。使用 items-end 确保所有元素底部对齐。下拉选择框（Select）的宽度应与输入框（Input）对齐，SelectTrigger 应使用 `w-full` 类名确保宽度一致。
